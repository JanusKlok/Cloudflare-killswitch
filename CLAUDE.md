# Project instructions

## Changelog

When making any notable code change (new feature, bug fix, behaviour change, removal), add a bullet to the `[Unreleased]` section of `CHANGELOG.md` before finishing the task. When a version is released, the `[Unreleased]` section gets renamed to `[x.y.z] — YYYY-MM-DD` and a fresh empty `[Unreleased]` block is added above it.
